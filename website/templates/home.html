Project: Premier League Analytics Dashboard

A Case Study in Data Storytelling, UX Design, and Season Aware Intelligence
Project Overview

I decided to combine two of my passions – data storytelling and my love of football by developing an interactive Power BI dashboard to analyze English Premier League performance across multiple seasons. The project focused on solving real-world modelling challenges, including non-standard date structures, ranking logic, and interactive reporting.

The goal was to build a professional grade reporting experience that feels as polished as a commercial sports analytics product — not just a collection of visuals. All data for this project was sourced from Football-Data.co.uk.

An early challenge to this project was that football seasons don’t follow a traditional calendar structure – they span two calendar years (eg. 2019-2020). This creates complications when applying standard time intelligence functions.

The project demonstrates my ability to:

  • Model complex sports data using a clean star schema
  • Engineer season aware time intelligence
  • Create dynamic, user friendly visuals
  • Integrate branding elements like team logos and contextual tooltips
  • Build scalable, maintainable Power BI solutions
It also serves as a flagship example of how I combine analytics, design, and product thinking — the same blend I bring to music, ticketing, and event focused analytics roles.

Data Architecture & Modeling

Star Schema Foundation

I designed the model around a disciplined star schema to ensure clarity, performance, and scalability. Key components include:

  • Fact tables for match results and season performance
  • Dimension tables for teams, dates, seasons, and competitions
  • Clean relationship paths to avoid ambiguity
  • Slim, efficient fact tables that support fast DAX calculations

Season Aware Time Intelligence

Football seasons span two calendar years, which breaks traditional time intelligence. To solve this, I built a custom framework that:

  • Defines seasons independently of calendar years
  • Supports filtering by season, match-week, or date
  • Uses calculation items to simplify repeated logic
  • Ensures measures behave correctly regardless of slicer combinations

This enables accurate metrics such as:

  • Season to date performance
  • Rolling form over the last N matches
  • Dynamic league table rankings
  • Cross season comparisons

This framework is one of the most technically sophisticated parts of the project and a reusable asset for future sports analytics work.


Key Measures

Some of the core metrics include:

  • Wins, draws, losses
  • Points and points per match
  • Goals for/against and goal difference
  • Form over recent matches
  • Season aggregates and rolling metrics

League standings were calculated using a custom RANKX measure that incorporates multiple tie-breakers – Points, Goal Difference, and Wins. By combining these into a weighted scoring model, I ensure rankings remained deterministic – even when teams were tied on primary metrics.

Branding & Visual Identity

Bent Canoe Fermentation Sales

PowerBI | UX Design | Python | Prompt Engineering

A simple interactive Power BI dashboard showing web sales for a fictional brewery. Bold, contrasting colours that tie in company branding. Important KPIs are easily identifiable.

Data Creation

I built this project to showcase design, Power BI, prompt engineering, and Python skills by simulating sales for a fictional brewery’s online store. I created a full product catalog with categories, variants, and pricing — over 200 SKUs inspired by real brewery offerings.

To generate realistic sales activity for Q3 2025, I used AI prompting to produce randomized transactions. A Python script handled the heavy lifting: it weighted popular items like Beer and Ready-To-Drink products more heavily and boosted weekend sales to mimic real buying patterns. The script outputs a parquet file, which stores large datasets efficiently and loads quickly into Power BI.

{% verbatim %} 


import pandas as pd
import numpy as np
from datetime import datetime, timedelta
import random

# -----------------------------
# Load SKU catalog
# -----------------------------
sku_df = pd.read_csv(
    r"C:\Users..BreweryProducts.csv", encoding="latin1"
)


# Normalize column names
sku_df.columns = sku_df.columns.str.lower().str.replace(" ", "_")

# -----------------------------
# Helper functions
# -----------------------------

def base_daily_demand(category):
    """Return a baseline expected number of transactions per day by category."""
    if category == "Beer":
        return np.random.randint(20, 40)
    if category == "Ready To Drink":
        return np.random.randint(15, 30)
    if category == "Spirit":
        return np.random.randint(2, 6)
    if category == "Energy":
        return np.random.randint(10, 20)
    if category == "Hydration":
        return np.random.randint(8, 15)
    if category == "Wellness":
        return np.random.randint(5, 10)
    if category == "Non-Alcoholic":
        return np.random.randint(6, 12)
    return 5

def weekend_multiplier(date):
    """Increase demand on weekends."""
    if date.weekday() in [4, 5]:  # Friday, Saturday
        return 1.8
    if date.weekday() == 6:       # Sunday
        return 1.4
    return 1.0

def quantity_for_variant(variant):
    """Assign realistic quantities per transaction."""
    if "1 Can" in variant or "1 Bottle" in variant:
        return np.random.randint(1, 4)
    if "4-Pack" in variant or "4-Can" in variant:
        return np.random.randint(1, 2)
    if "6-Pack" in variant:
        return np.random.randint(1, 2)
    if "12-Pack" in variant:
        return 1
    if "24-Case" in variant:
        return 1
    return 1

# -----------------------------
# Generate Q3 2025 date range
# -----------------------------
start = datetime(2025, 7, 1)
end = datetime(2025, 9, 30)
dates = pd.date_range(start, end)

# -----------------------------
# Generate transactions
# -----------------------------
transactions = []

transaction_counter = 1

for date in dates:
    for _, row in sku_df.iterrows():

        # Determine expected demand
        base = base_daily_demand(row["category"])
        multiplier = weekend_multiplier(date)
        expected = int(base * multiplier)

        # Random chance that SKU has zero sales today
        if random.random() < 0.65:
            continue

        # Number of transactions for this SKU today
        num_tx = np.random.poisson(expected / 20)

        for _ in range(num_tx):
            qty = quantity_for_variant(row["variant_size"])
            price = row["price"]
            total = round(price * qty, 2)

            transactions.append({
                "transaction_id": f"T{transaction_counter:06d}",
                "transaction_date": date.strftime("%Y-%m-%d"),
                "sku": row["sku"],
                "product_name": row["name"],
                "category": row["category"],
                "variant_size": row["variant_size"],
                "unit_price": price,
                "quantity": qty,
                "total_amount": total
            })

            transaction_counter += 1

# Convert to DataFrame
transactions_df = pd.DataFrame(transactions)

# -----------------------------
# Save or inspect
# -----------------------------
print(transactions_df.head())
print(f"Generated {len(transactions_df):,} transactions.")
transactions_df.to_parquet("transactions_q3_2025.parquet", index=False)
